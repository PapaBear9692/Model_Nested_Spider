## How to Enable 100 PTM Mode

### Method 1: Environment Variable (Recommended)

```bash
# Windows PowerShell
$env:PTM100="yes"; python trainer.py --num_learnware 100 ...

# Windows CMD
set PTM100=yes && python trainer.py --num_learnware 100 ...

# Linux/Git Bash
PTM100=yes python trainer.py --num_learnware 100 ...
```

### Method 2: Import in Code

```python
# At the top of your script
import os
os.environ['PTM100'] = 'yes'

# Then import learnware components
from learnware.learnware_info_100 import BKB_100_SPECIFIC_RANK, MODEL_100_2FEAT_DIM
```

## Data Structure

### Prototype .pkl File Format

```python
[
    # x[0]: Uniform features (from Swin Transformer)
    torch.Tensor([num_classes, 3072]),

    # x[1]: Heterogeneous features (from 100 PTMs)
    {
        'resnet18': torch.Tensor([num_classes, 512]),
        'resnet50': torch.Tensor([num_classes, 2048]),
        'efficientnet_b0': torch.Tensor([num_classes, 1280]),
        'vit_b_16': torch.Tensor([num_classes, 768]),
        # ... 100 models total
    },

    # x[2]: Ground-truth rankings (optional)
    torch.Tensor([100])  # ranking of all 100 PTMs
]
```

## Commands
### Install timm (required for downloading 35 models)
```
pip install timm
```

### Generate Prototype Data

```bash
python tools/generate_prototype_100.py \
    --dataset CIFAR10 \
    --data_path ./data \
    --output_path ./required_files/data/implclproto_100 \
    --num_classes 10 \
    --batch_size 16
```

### Training with 100 PTMs

```bash
# Windows PowerShell
$env:PTM100="yes"; python trainer.py \
    --seed 0 \
    --train_dataset c86 c59 c16 c14 c38 c43 c9 c12 c32 c19 c31 c57 c29 \
    --val_dataset c86 \
    --test_dataset CIFAR10 Caltech101 DTD Pet Aircraft CIFAR100 Cars SUN397 dSprites \
    --data_sub_url swin_base_7_checkpoint \
    --heterogeneous \
    --num_learnware 100 \
    --batch_size 4 \
    --data_url "D:\Study\Thesis\Model_Nested_Spider\required_files\data\implclproto_100" \
    --log_url "D:\Study\Thesis\Model_Nested_Spider\required_files\log_100" \
    --num_workers 0
```

## Key Differences: 10 PTM vs 100 PTM Mode

| Aspect | 10 PTM Mode | 100 PTM Mode |
|--------|-------------|--------------|
| `--num_learnware` | 10 | 100 |
| Data folder | `implclproto/` | `implclproto_100/` |
| Env Variable | (none) | `PTM100=yes` |
| Batch Size | 16 | 4-8 (more memory) |
| Model Prompts | 10 | 100 |
| Training Time | Baseline | ~10x slower |
| Memory Usage | ~4GB | ~8-12GB |

---

## Memory Optimization Tips

For 100 PTM mode with limited GPU memory:

1. **Reduce batch size**: `--batch_size 4` or even `--batch_size 2`
2. **Use gradient accumulation**: Effective batch = batch_size × accum_steps
3. **Enable mixed precision**: Add `--fp16` flag (if supported)
4. **Reduce heterogeneous samples**: `--heterogeneous_sampled_maxnum 20`

## Troubleshooting

### Error: `KeyError: 'resnet34'`
- **Cause**: Prototype data file doesn't have features for all 100 PTMs
- **Solution**: Regenerate prototype data using `generate_prototype_100.py`

### Error: `CUDA out of memory`
- **Cause**: 100 models × large feature dimensions = high memory
- **Solution**: Reduce `--batch_size` to 4 or 2

### Error: `ModuleNotFoundError: No module named 'timm'`
- **Cause**: Some models require timm library
- **Solution**: `pip install timm`
